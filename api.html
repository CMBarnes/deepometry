API¶

Parsing and Loading¶

deepometry.parse.parse(paths, output_directory, size, channels=None)[source]¶

Prepare a collection of images for training, evaluation, and prediction.

.TIF/.TIFF data is expected to be single-channel data with channel information encoded in the filename: e.g., /.*_Ch[0-9]+.TIF/.

Parameters:
  • paths – List of pathnames to parse.
  • output_directory – Location to exported parsed image data.
  • size – Final dimension (size, size) of parsed image data.
  • channels – List of channels to export. Use None to export all available channels.
deepometry.utils.load(directories, convert=True, sample=None)[source]¶

Load image and label data.

Parameters:
  • directories – List of directories. Subdirectories of directories are class labels and subdirectory contents are image data as NPY arrays.
  • convert – Convert label strings to integers (default: True).
  • sample – Undersample image data per subdirectory (default: None).
Returns:

(x, y, units) where x is concatenated image data of shape (N samples, row, col, channels), y is a list of labels of length N samples, and units is the number of unique labels.

Model¶

class deepometry.model.Model(shape, units, directory=None, name=None)[source]¶

Create a model for single-cell image classification.

Parameters:
  • shape – Input image shape, including channels. Grayscale data should specify channels as 1. Check your keras configuration for channel order (e.g., "image_data_format": "channels_last"). Usually, this configuration is defined at $HOME/.keras/keras.json, or %USERPROFILE%\.keras\keras.json on Windows.
  • units – Number of predictable classes.
  • directory – (Optional) Output directory for model checkpoints, metrics, and metadata. Otherwise, the package’s data directory is used.
  • name – (Optional) A unique identifier for referencing this model.

evaluate(x, y, batch_size=32, verbose=0)[source]¶

Compute the loss value & metrics values for the model in test mode.

Computation is done in batches.

Parameters:
  • x – NumPy array of test data.
  • y – NumPy array of target data.
  • batch_size – Number of samples evaluated per batch.
  • verbose – Verbosity mode, 0 = silent, or 1 = verbose.
Returns:

Tuple of scalars: (loss, accuracy).
extract(x, batch_size=32, standardize=False, verbose=0)[source]¶

Extract learned features from the model.

Computation is done in batches.

Parameters:
  • x – NumPy array of data.
  • batch_size – Number of samples evaluated per batch.
  • standardize – If True, center to the mean and component wise scale to unit variance (default: False).
  • verbose – Verbosity mode, 0 = silent, or 1 = verbose.
Returns:

NumPy array of shape (samples, features).
fit(x, y, batch_size=32, class_weight='auto', epochs=512, validation_split=0.2, verbose=0)[source]¶

Train the model for a fixed number of epochs (iterations on a dataset). Training will automatically stop if the validation loss fails to improve for 20 epochs.

Parameters:
  • x – NumPy array of training data.
  • y – NumPy array of target data.
  • batch_size – Number of samples per gradient update.
  • class_weight – Dictionary mapping labels to weights. Use "auto" to automatically compute weights. Use None to ignore weights.
  • epochs – Number of times to iterate over the training data arrays.
  • validation_split – Fraction of the training data to be used as validation data.
  • verbose – Verbosity mode. 0 = silent, 1 = verbose, 2 = one log line per epoch.
